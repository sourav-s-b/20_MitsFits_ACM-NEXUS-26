# NexusPath Progress — 01:00

## Bug Fixes & Stabilization: The "Multi-Fleet" Patch
During large-scale multi-tenant stress testing, several core engine instabilities were observed and successfully patched:

### 1. Vector Mapping Collision Fixed
- `start_shipment()` inside `main_routes.py` successfully calculated custom origin-to-destination paths via TomTom, but failed to persist the new `Destination` vector to the in-memory Redis equivalent. All trucks internally believed they were routing to Bangalore. **Fixed: Explicit geometric dictionary overrides applied.**

### 2. Simulator Index Out of Bounds Crash
- `decision_routes.py`'s `confirm_reroute` function pulled TomTom points structured as `[{latitude: X, longitude: Y}]`, while the backend simulator strictly expects `[{lat: X, lon: Y}]`. This mismatch instantly zeroed out the truck's position upon reroute. **Fixed: Implemented list comprehension sanitization over the raw JSON payload.**

### 3. Absolute Multi-Truck Rendering
- React's `App.jsx` was previously restricted to only rendering the primary active WebSocket's geometry on map (`currentShipmentId`). **Fixed:** Upgraded `fleet` state to the `<App />` root and wired a secondary polling mechanism. The Leaflet Map layer now actively iterating and rendering `fleet.map()`, visualizing the entire array of trucks moving independently around the world simultaneously!
