# NexusPath Progress — 01:25

## React UI Optimization & Routing Deviance Polishing
During high-scale multi-truck visual logic testing, two secondary critical bugs were detected and eliminated.

### 1. Fix: Extreme React-Leaflet Map Lag (SVG Thrashing)
**Issue:** The WebSockets transmit truck coordinate data (which includes a massive 5000+ items `route` array) every 3 seconds to keep the simulator alive. React's functional component parsed all 5,000 elements over and over, forcing Leaflet to rebuild massive `<Polyline>` DOM SVG nodes unnecessarily.
**Solution:** Intercepted the massive arrays cleanly via React's `useMemo` hook against dependencies like `[shipment.route?.length]`. Array logic only recalculates explicitly if topological structure changes, drastically slicing 2.5s rendering overhead per tick down to sub-10ms.

### 2. Fix: Duplicate "Same Path" Reroute Algorithms
**Issue:** When pulling shadow routes from TomTom using `maxAlternatives=2` against heavy traffic, TomTom frequently returned alternate lanes on the exact same road, which rendered directly atop the `<Polyline>`, preventing the user from physically seeing the alternative.
**Solution:** Hardcoded `minDeviationDistance=5000` & `alternativeType=anyRoute` into the `calculateRoute` query path within `orchestration_routes.py`. It forces the routing algorithm to seek paths physically scattered by at least 5 kilometers from the primary artery, enforcing visually distinct pathing lines on the map UI.
