## React Error Boundary Hardening 
Discovered a critical state mismatch caused by asynchronous backend/frontend server reboots that resulted in an unexpected UI blackout crash.

### 1. Fix: React DOM Fatal Black Screen Crash
**Issue:** When the Python backend was running a stale memory instance (without auto-reload), pulling an old TomTom API definition mapped alternative routes as `[{"latitude": X, "longitude": Y}]` instead of `[{"lat": X, "lon": Y}]`. When this stale data was pushed to the frontend, `App.jsx` mapped `undefined` directly into Leaflet's core SVG engine, triggering a fatal catastrophic failure that entirely crashed the React application layout frame (turning the screen black).
**Solution:** Hardcoded nullish coalescing operators `??` recursively across the `mainRoutePoints` payload parser inside `App.jsx`. The frontend is now fully resilient to data structure drift and legacy cache schemas, parsing legacy data interchangeably without crashing Leaflet natively.
